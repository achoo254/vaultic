Phase 2 — Crypto Core (Rust)

CriticalNo screens

vaultic-crypto crate: Argon2id KDF, AES-256-GCM encrypt/decrypt, HKDF key derivation, password generator. Test vectors for WebCrypto interop.

Functions

derive_master_key, derive_encryption_key, derive_auth_hash, encrypt, decrypt, generate_password

Security

No panics (Result<T>), no unsafe, zeroize keys, constant-time compare

Output

test-vectors.json for Phase 4 WebCrypto interop testing

Phase 3 — API Server & Database

CriticalNo screens

Axum REST API: auth (register/login/JWT), sync relay (opt-in push/pull), share broker, purge endpoint. Server stores only encrypted blobs.

Auth Endpoints

POST /auth/register, /auth/login, /auth/refresh

Sync Endpoints (opt-in)

POST /sync/push, GET /sync/pull, DELETE /sync/purge

Share Endpoints

POST /share, GET /share/:id, DELETE /share/:id

DB Tables

users, vault_items (sync copy), folders, secure_shares

Tech Stack

Monorepo: Cargo workspace (4 Rust crates) + Turborepo (7 pnpm packages).

Rust Crates

vaultic-crypto — Argon2id, AES-256-GCM, HKDF

vaultic-server — Axum API (auth+sync+share)

vaultic-types — Shared types (serde)

vaultic-migration — SeaORM migrations

TS Packages (@vaultic/*)

types — Shared TS types

crypto — WebCrypto bridge

storage — VaultStore + IndexedDB

sync — Sync engine + LWW

api — Server API client (ofetch)

ui — React + shadcn/ui + Tailwind

extension — WXT (Chrome+Firefox)

Server

Framework: Axum (Rust)

ORM: SeaORM + PostgreSQL 16

TLS: rustls (not openssl)

Auth: JWT (access+refresh)

Extension

Framework: WXT + React

State: Zustand

Storage: IndexedDB (idb)

Crypto: WebCrypto + argon2-browser

Infrastructure

CI/CD: GitLab CI (gitlabs.inet.vn)

Registry: gitlabs.inet.vn:5050

Deploy: PM2 on CentOS 7

License: MIT

Architecture

Offline-first: all CRUD local

Cloud Sync: opt-in (user toggle)

Zero-knowledge: server sees ciphertext only

Share: independent of sync

Sync off: delete or keep frozen
